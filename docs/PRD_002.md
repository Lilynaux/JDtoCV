ResumeTailor V1.1 PRD Update

Feature

Multi-LLM Provider Support

⸻

Background

当前 ResumeTailor 已接入 OpenAI API。

为了降低开发和测试成本，需要支持多个 LLM Provider。

开发阶段默认使用 Gemini。

未来可无缝切换至 GPT。

⸻

Goal

实现模型提供商可配置化。

用户无需修改业务代码。

仅通过配置文件即可切换模型。

⸻

Requirements

Supported Providers

Gemini

Provider:

google

Models:

* gemini-2.5-flash
* gemini-2.5-pro

用途：

默认开发模型

⸻

OpenAI

Provider:

openai

Models:

* gpt-5
* gpt-5-mini

用途：

最终简历优化

高级英文润色

Cover Letter

⸻

Configuration

新增：

config.yaml

Example:

llm:
  provider: google
  model: gemini-2.5-flash

切换 GPT：

llm:
  provider: openai
  model: gpt-5

无需修改代码。

⸻

Environment Variables

.env

OPENAI_API_KEY=
GEMINI_API_KEY=

系统根据 provider 自动选择对应 API Key。

⸻

Architecture

新增统一接口：

BaseLLM

class BaseLLM:
    def generate(
        self,
        prompt: str
    ) -> str:
        pass

⸻

OpenAIProvider

class OpenAIProvider(BaseLLM):

⸻

GeminiProvider

class GeminiProvider(BaseLLM):

⸻

业务模块统一调用：

llm.generate(prompt)

禁止业务代码直接调用：

client.chat.completions.create(...)

这样未来增加新模型时无需修改业务逻辑。

⸻

Affected Modules

JD Analyzer

Resume Optimizer

CN → EN Sync

Match Report

Cover Letter (future)

Interview Generator (future)

全部通过统一 LLM Interface 调用。

⸻

Default Behavior

首次安装：

provider: google
model: gemini-2.5-flash

作为默认配置。

原因：

* 免费额度大
* 成本低
* 开发阶段足够使用

⸻

Error Handling

如果模型调用失败：

自动重试一次。

失败后提示：

Provider: Gemini
Model: gemini-2.5-flash
Error: API unavailable

不允许程序直接崩溃。

⸻

Future Compatibility

预留 Provider 扩展能力：

* DeepSeek
* OpenRouter
* Claude
* Qwen

新增 Provider 时：

仅需增加：

providers/

├── openai.py
├── gemini.py
├── deepseek.py
└── qwen.py

业务代码无需修改。

⸻

Acceptance Criteria

✓ Gemini 可以正常运行全部功能

✓ OpenAI 可以正常运行全部功能

✓ 修改 config.yaml 即可切换模型

✓ 默认模型为 gemini-2.5-flash

✓ 业务模块不感知底层模型差异

✓ 后续新增模型无需重构项目

顺便建议 CC 再加一个很小但未来很有用的配置：

llm:
  provider: google
  model: gemini-2.5-flash
  temperature: 0.3
  max_tokens: 4000

因为简历优化和 JD 分析属于低创造性任务，temperature 以后你大概率会长期固定在 0.2-0.4 之间。这样以后切 GPT、DeepSeek、Claude 时参数也统一。
