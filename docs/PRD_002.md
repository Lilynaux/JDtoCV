# PRD_002: Multi-LLM Provider Support（多模型供应商支持）

> 增量需求（V1.1），在 [PRD_001](./PRD_001.md) 的基础上扩展，不重写整体产品需求。

## **1. Feature Overview（功能概述）**

为 ResumeTailor 增加多 LLM Provider 支持。

---

## **2. Background（背景）**

当前 ResumeTailor 已接入 OpenAI API。为了降低开发和测试成本，需要支持多个 LLM Provider：开发阶段默认使用 Gemini，未来可无缝切换至 GPT。

---

## **3. Goal（目标）**

实现模型提供商可配置化：用户无需修改业务代码，仅通过配置文件即可切换模型。

---

## **4. Requirements（需求）**

### Supported Providers（支持的供应商）

| Provider | 配置值 | 模型 | 用途 |
| --- | --- | --- | --- |
| Gemini | `google` | `gemini-2.5-flash` / `gemini-2.5-pro` | 默认开发模型 |
| OpenAI | `openai` | `gpt-5` / `gpt-5-mini` | 最终简历优化、高级英文润色、Cover Letter |

---

## **5. Configuration（配置方式）**

新增 `config.yaml`：

```yaml
llm:
  provider: google
  model: gemini-2.5-flash
```

切换至 GPT，无需修改代码：

```yaml
llm:
  provider: openai
  model: gpt-5
```

---

## **6. Environment Variables（环境变量）**

`.env`：

```bash
OPENAI_API_KEY=
GEMINI_API_KEY=
```

系统根据 `provider` 自动选择对应 API Key。

---

## **7. Architecture（架构设计）**

新增统一接口 `BaseLLM`：

```python
class BaseLLM:
    def generate(self, prompt: str) -> str:
        pass


class OpenAIProvider(BaseLLM):
    ...


class GeminiProvider(BaseLLM):
    ...
```

业务模块统一调用 `llm.generate(prompt)`，禁止业务代码直接调用 `client.chat.completions.create(...)` 等供应商专属接口。这样未来增加新模型时无需修改业务逻辑。

---

## **8. Affected Modules（受影响模块）**

全部通过统一 LLM Interface 调用：

- JD Analyzer
- Resume Optimizer
- CN → EN Sync
- Match Report
- Cover Letter（future）
- Interview Generator（future）

---

## **9. Default Behavior（默认行为）**

首次安装默认配置：

```yaml
llm:
  provider: google
  model: gemini-2.5-flash
```

原因：

- 免费额度大
- 成本低
- 开发阶段足够使用

---

## **10. Error Handling（错误处理）**

模型调用失败时自动重试一次；失败后提示具体的 Provider / Model / Error，不允许程序直接崩溃。例如：

```text
Provider: Gemini
Model: gemini-2.5-flash
Error: API unavailable
```

---

## **11. Future Compatibility（未来兼容性）**

预留 Provider 扩展能力（DeepSeek、OpenRouter、Claude、Qwen）。新增 Provider 时仅需在 `providers/` 下新增对应文件，业务代码无需修改：

```text
providers/
├── openai.py
├── gemini.py
├── deepseek.py
└── qwen.py
```

---

## **12. Acceptance Criteria（验收标准）**

- [x] Gemini 可以正常运行全部功能
- [x] OpenAI 可以正常运行全部功能
- [x] 修改 `config.yaml` 即可切换模型
- [x] 默认模型为 `gemini-2.5-flash`
- [x] 业务模块不感知底层模型差异
- [x] 后续新增模型无需重构项目

---

## **13. Appendix: Recommended Config（附录：推荐配置参数）**

简历优化和 JD 分析属于低创造性任务，建议长期固定 `temperature` 在 0.2–0.4 之间，未来切换 GPT / DeepSeek / Claude 时参数口径保持统一：

```yaml
llm:
  provider: google
  model: gemini-2.5-flash
  temperature: 0.3
  max_tokens: 4000
```
